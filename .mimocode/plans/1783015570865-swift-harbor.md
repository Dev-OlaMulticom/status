# Plan: Mejora de Interfaz e Integración WHM/Cloudflare

## Resumen
Mejorar la interfaz del dashboard y ampliar la integración con WHM y Cloudflare para obtener información completa de cuentas, dominios y zonas DNS.

---

## 1. Mejora del Provider Cloudflare (`src/providers/cloudflare.provider.ts`)

### Problema actual
Solo obtiene: lista de zonas (id, name, status, paused, plan, ssl) y A record IP por dominio.

### Mejoras a implementar

**Nuevas funciones:**
- `getAllDnsRecords(zoneId)` — Obtener TODOS los registros DNS de una zona (A, AAAA, CNAME, MX, TXT, NS, SRV, CAA)
- `getDnsSecStatus(zoneId)` — Obtener estado DNSSEC de la zona
- `getZoneSettings(zoneId)` — Obtener configuración clave de la zona (SSL mode, security level, min TLS, Always Use HTTPS, etc.)
- `getAccountInfo()` — Obtener información de la cuenta Cloudflare (email, ID)
- `listZonesWithDetails()` — Zonas expandidas con nameservers,_CREATED_ON, MODIFIED_ON, original_registrar, original_dnshost

### Nuevo tipo `CloudflareZoneInfo` ampliado:

## 2. Mejora del Provider WHM (`src/providers/whm.provider.ts`)

### Problema actual
Solo obtiene: lista de dominios (domain, user, status, type, main_domain, ip, addon, sub_domain) y conteo de emails.

### Mejoras a implementar

**Nueva función:**
- `getAccountDetails(usernames)` — Llamar al endpoint `listaccts` de WHM para obtener información detallada de cada cuenta

### Nuevo tipo `WhmAccountDetail`:
```typescript
export interface WhmAccountDetail {
  username: string
  domain: string
  plan: string              // Paquete contratado (ej: "VPS", "Shared", etc.)
  diskused: number          // Disco usado en MB
  diskquota: number         // Disco total en MB
  diskpercent: number       // Porcentaje de disco usado
  bwused: number            // Ancho de banda usado en MB
  bwquota: number           // Ancho de banda total en MB
  bwpercent: number         // Porcentaje de BW usado
  emailAccounts: number     // Número de cuentas de email
  suspended: boolean
  ip: string
  startdate: string | null  // Fecha de inicio de la cuenta
}
```

**Nueva función `extractAccountUsage()`:**
- Llamar `listaccts` con parámetros para obtener uso de disco, BW, y detalles del plan
- Combinar con datos existentes de `get_domain_info`
- Actualizar `WhmExtractResult` para incluir detalles de uso

### Archivos a modificar:
- `src/providers/whm.provider.ts` — Agregar `getAccountDetails()` y `extractAccountUsage()`
- `src/dto/whm.dto.ts` — Agregar `WhmAccountDetail` y actualizar `WhmExtractResult`
